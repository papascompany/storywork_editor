StoryWork 개발 계획 대시보드

1,000+개의 PNG 포즈 라이브러리(향후 SVG 병행), 관리자 등록 리소스, AI 대본 분석을 결합해 fabric.js 기반 캔바급 온라인 스토리보드 편집기를 만들고, 완성 페이지를 POD PDF로 출판하는 서비스. 본 문서는 첫 라인을 짜기 전 보아야 할 단일 진실 원천(SSOT) 시각화입니다.

📌 자산 현황 & 정책 (확정)

  • 보유: PNG 1,058개 — 750×750(96%), 알파 채널 100%, 매직바이트 위장 0건
  • 라이선스: data/poses/raw/LICENSE.json 1개 폴더 디폴트. holder=StoryWork, terms=all-rights, 상업 사용 OK
  • 키포인트: 사이드카 0건 → 최소 3점(head/mouth/center) 자동 추정. 손/발은 검수로 보강
  • 저해상도: 750px 자산은 lowDpi 태그 → 자동 배치 시 페이지 1/2 이하 슬롯에만 사용
  • SVG: 동일 파이프라인의 어댑터로 M10+ 입점 (ADR-0011)

일반 / 크리에이터(구독) / 관리자 / 커뮤니티

① 지금 결정해야 할 것 — 의사결정 체크리스트

개발을 시작하기 전 다음 9개를 확정해야 합니다. 기본값(권장)은 ★ 표시. 결정이 끝나면 체크박스를 표시한 채 PR/이슈에 첨부하세요.

D1. 운영 형태 / 도메인

사용자/관리자 도메인 분리 여부, 결제·법인 명의

분리는 보안(2FA·세션·CSP) + 빌드 최적화 이점.

D2. 인증/계정

RLS 통합·운영 단순. 필수 소셜 1개 + 이메일 OTP 권장.

D3. 결제·구독

초기엔 Stripe Checkout으로 빠르게. 한국 결제는 M7~M8 사이 추가.

D4. 가격 정책 / 플랜 라인업

"마이데이터" 구독 핵심 가치 → 월 정기 권장. 가격 숫자 확정 필요.

D5. 인쇄 사양 (POD)

제휴 인쇄소가 없다면 B5/A5 표준+3mm bleed로 시작.

D6. AI 비용 한도 / 모델

한도 = 비용 폭주 방지. 사용량은 PostHog로 가시화.

D7. 호스팅

속도 우선 → Vercel. 비용 폭주 우려가 명백해지면 워커만 분리.

D8. 포즈 자산 정책 확정

자산 1,058개 인입 준비 완료. 인입 후 검수 큐에서 키포인트 점진 보강.

D9. 협업/오토파일럿 강도

초기엔 ON 권장. 신뢰도 쌓이면 점진적으로 풉니다.

표준 흐름 (자동 배치 기능 신규 개발 예시)

  1. 휴먼: /sw-next 호출
  2. orchestrator: roadmap.md에서 다음 작업 1건 픽 (예: M4-03)
  3. orchestrator → layout-composer: 자기완결 프롬프트로 위임
  4. layout-composer: 알고리즘 + 단위 테스트
  5. layout-composer → qa-tester: 골든셋 확장
  6. qa-tester: simplify + security-review 자동 실행
  7. orchestrator: PR(draft) + roadmap 체크 갱신
  8. 휴먼: 리뷰 → 머지 → 다음 호출 대기

⑪ 성능·접근성·보안 게이트

성능

  • 편집기 TTI ≤ 2.5s (P75 4G)
  • 200 객체 60/30 fps
  • PDF 16p ≤ 6s
  • API p95 ≤ 500ms
  • 모듈 bench 회귀 ±10% 이내

접근성

  • WCAG 2.1 AA
  • 키보드 단독 편집 가능
  • 대비 4.5:1 / 아이콘 3:1
  • ARIA Live 로 캔버스 변경 안내
  • prefers-reduced-motion 존중

보안

  • SVG sanitize (script/외부 ref/foreignObject)
  • 업로드 MIME 화이트리스트 + ClamAV
  • RLS 우회 0 (서버 액션 검증)
  • SSRF 가드(외부 fetch 화이트리스트)
  • 관리자 2FA 강제
