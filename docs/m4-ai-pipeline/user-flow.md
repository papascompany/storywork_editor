# M4 AI Pipeline — 사용자 흐름

## 전체 7단계 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│  사용자 흐름: 대본 → 완성 스토리보드                               │
└─────────────────────────────────────────────────────────────────┘

1. 대본 입력
   ┌──────────────────────────────────────────────────────────┐
   │  사용자: 텍스트 박스에 대본 붙여넣기 또는 파일 업로드         │
   │  지원 형식: 소설 / 스크린플레이 / 에세이 / 일기 / 라이트노벨  │
   └──────────────────────────────────────────────────────────┘
   ↓

2. 형식 감지 + 장면 분석 (ai-script)
   ┌──────────────────────────────────────────────────────────┐
   │  @storywork/ai-script analyze()                          │
   │  - 형식 자동 감지 (rule-based)                            │
   │  - 장면 분리 + 대사 추출 + 화자 감지                       │
   │  - 감정/카메라앵글/분위기 메타 추출 (LLM 보강 시)           │
   │  - 등장인물 목록 + 멘션 수 집계                            │
   │  출력: SceneDoc (N 장면)                                  │
   └──────────────────────────────────────────────────────────┘
   ↓

3. 포즈/배경 추천 (ai-recommend)
   ┌──────────────────────────────────────────────────────────┐
   │  @storywork/ai-recommend                                  │
   │  - 장면 감정/시점 → 포즈 후보 K=5 (임베딩 + 룰)           │
   │  - 장면 배경/시간 → 배경 후보 K=3                         │
   │  - 캐릭터 ID 기반 포즈 일관성 유지                         │
   │  - lowDpi 자산은 작은 슬롯에만 배치 (ADR-0011a)           │
   └──────────────────────────────────────────────────────────┘
   ↓

4. 자동 배치 (ai-layout)
   ┌──────────────────────────────────────────────────────────┐
   │  @storywork/ai-layout compose()                          │
   │  - 판형 슬롯 매핑                                         │
   │  - 충돌 해소 (포즈 겹침 / safe-area 침범)                  │
   │  - 말풍선 + 대사 자동 배치                                │
   │  - seed 고정 → 결정론 보장                               │
   │  출력: fabricJson 초안 (N 페이지)                         │
   └──────────────────────────────────────────────────────────┘
   ↓

5. 편집기 후편집
   ┌──────────────────────────────────────────────────────────┐
   │  @storywork/editor-core (fabric.js)                      │
   │  - 자동 배치 결과를 편집기에 로드                           │
   │  - 포즈 교체: 한 클릭 alternatives 패널                   │
   │    (confidence + 대안 K=5 표시)                          │
   │  - 레이어 이동 / 크기 조정 / 텍스트 편집                    │
   │  - autosave 2초 디바운스                                  │
   └──────────────────────────────────────────────────────────┘
   ↓

6. PDF 출판 (pdf-engine)
   ┌──────────────────────────────────────────────────────────┐
   │  @storywork/pdf-engine                                    │
   │  - POD 인쇄 사양 PDF 생성                                 │
   │  - 재단선 / safe-area / 블리드 가이드 포함                 │
   │  - PublishJob 큐 (Inngest)                               │
   └──────────────────────────────────────────────────────────┘
   ↓

7. 소셜 공유 (M8)
   ┌──────────────────────────────────────────────────────────┐
   │  SNS 공유 / 공모전 출품 / 뽐내기 갤러리                     │
   └──────────────────────────────────────────────────────────┘
```

## 현재 구현 상태 (2026-06-03)

| 단계 | 상태 | 패키지 |
|---|---|---|
| 1. 대본 입력 | UI 미구현 (M4-04) | - |
| 2. 형식 감지 + 분석 | ✅ **구현 완료** (M4-01) | `@storywork/ai-script` |
| 3. 포즈/배경 추천 | 스캐폴드만 (M4-02) | `@storywork/ai-recommend` |
| 4. 자동 배치 | 스캐폴드만 (M4-03) | `@storywork/ai-layout` |
| 5. 후편집 | ✅ 편집기 동작 (M1) | `@storywork/editor-core` 등 |
| 6. PDF 출판 | 스캐폴드만 (M6) | `@storywork/pdf-engine` |
| 7. 소셜 | 커뮤니티 페이지만 (M8) | - |

## 결정론 보장

- `seed` 파라미터로 동일 입력 → 동일 출력
- temperature 0 (LLM 활성 시)
- alternatives K=5: 사용자에게 다음 후보 노출
- CI에서는 `STORYWORK_LLM=0` (룰-only, 비용 없음)
